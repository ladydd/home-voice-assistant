# 项目 B：ESP32 无线麦克风（学习项目）

> 目标：**学习 ESP32 开发**。用 XIAO ESP32-S3 Sense 做一个无线拾音终端，
> 替代项目 A 里那根 USB 麦克风线，让麦克风可以放在家里任意位置。
> 特点：要写固件、循序渐进、重点是学到嵌入式核心技能。

---

## 定位说明

这不是为了"工程最优"——功能上项目 A 已经够了。
这个项目的价值是**练手 ESP32**：

- I2S 数字麦克风驱动
- WiFi 联网（STA 模式）
- HTTP 通信
- 音频采集与传输
- 按键/触发逻辑
- 为后续更复杂的嵌入式项目打基础

板子作为无线终端的实际好处：可放客厅/床头/厨房，不用拉线到服务器。

---

## 目标架构

```
┌─────────────┐  WiFi/局域网   ┌──────────────────────────┐
│  ESP32-S3   │  录音上传       │      Hermes 服务器         │
│  (麦克风)    │ ────────────► │  STT → LLM → TTS          │
└─────────────┘               │              └──► 🔊 家庭音响 │
                              └──────────────────────────┘
```

- **板子**：只负责"听"——麦克风录音 + WiFi 上传。端侧极简。
- **服务器**："想"+"说"一条龙（复用项目 A 已经跑通的 Hermes 语音链路）。
- **音响**：接服务器出声（3.5mm 线 或 蓝牙）。

> 关键：项目 B 复用项目 A 的服务端。板子只是替换掉那个 USB 麦克风。

## 硬件现状

| 设备 | 状态 | 作用 |
|------|------|------|
| XIAO ESP32-S3 Sense | ✅ 已有 | 录音 + WiFi 联网 |
| 板载 PDM 麦克风 (MSM261D3526H1CPM) | ✅ 自带 | 拾音 |
| Hermes 服务器 | ✅ 已有 | 服务端（项目 A 已搭好） |
| 喇叭 | — | **不需要**（出声由服务器+音响负责） |

> 注意：ESP32-S3 只有 BLE，没有经典蓝牙，连不了家里蓝牙音响。
> 但本项目板子不负责出声，所以这个限制不影响。

## 板载麦克风效果评估（按经验）

板载 PDM 麦克风定位是"能用"，不是"好用"。

| 场景 | 效果预估 |
|------|---------|
| 安静环境 + 30cm 内说话 | ✅ 没问题，STT 识别率高 |
| 有背景噪音（电视、空调） | ⚠️ 凑合，偶尔识别错 |
| 音响正在播放 + 同时说话 | ❌ 会录进回声，干扰 STT（按键模式可天然规避）|
| 隔一两米远喊话 | ⚠️ 声音偏小，看运气 |

- 单麦克风、无降噪、无回声消除（AEC）
- 近场语音识别够用，别当远场麦克风
- **升级路线**：后续可外接 I2S 数字麦克风（INMP441/SPH0645，音质更好）

## 触发方式（板子何时录音/上传）

这是核心交互问题，几种选择：

| 方案 | 说明 | 评价 |
|------|------|------|
| 按键说话（对讲机） | 按住录音，松开上传 | ✅ 最简单不误触，建议 v1 |
| 语音唤醒词 | 喊唤醒词激活 | 体验好但要跑模型、费电、复杂 |
| 语音活动检测 VAD | 检测到说话自动录 | 不用按，但噪音会误触发 |
| 服务器下发指令 | 服务器通知"该你说了" | 适合多轮对话，第一句仍需别的触发 |

> 板子上有 BOOT/RESET 按钮（很小，在 USB 口侧），后续可外接大按钮。
> v1 甚至可以先用"固定文字 / 串口输入"触发，连按钮都不用，先打通链路。

最终形态设想：唤醒词触发第一句 → 之后 VAD 自动续话 → 音响播完等下一句。
（这是后话，v1 先按键）

## 分阶段实施计划（每阶段可独立验证）

> 原则：先打通最短的文字闭环，再逐步加音频。每步都能单独看到结果。

### Phase 0 — 连通性冒烟测试 ⭐ 先做这个
**目标**：板子能 WiFi 访问服务器，服务器能调通 Hermes，文字一来一回。
```
板子(发写死的文字) ──HTTP──► 中转服务 ──► Hermes ──► 文字回复 ──► 板子(串口打印)
```
- 不涉及麦克风/音频，纯验证网络链路
- 串口能看到 Hermes 回复 = 链路通

### Phase 1 — 麦克风采集验证
**目标**：确认板载麦克风能录到清晰音频。
- 板子录一段 → 存 WAV → 拉到 Mac 播放确认
- 纯硬件验证，与服务器无关

### Phase 2 — 音频上传 + STT
**目标**：板子录音上传，服务器转文字后送 Hermes。
```
板子(录音) ──► 中转服务(STT) ──► 文字 ──► Hermes ──► 文字回复
```

### Phase 3 — 完整闭环（出声）
**目标**：Hermes 回复 → TTS → 家庭音响播放。
```
板子(录音) → 服务器(STT→LLM→TTS) → 音响出声
```
- 板子端几乎不用改（出声在服务器侧）
- 复用项目 A 的播放链路

## 技术选型

### 板子端固件
- 框架：**v1 建议先用 Arduino**（轻量、上手快，Phase 0 只需 WiFi+HTTP）
  - 后续涉及 I2S 深度控制可转 **ESP-IDF**（驱动支持最全）
- 麦克风：I2S PDM 模式
  - 引脚待确认（XIAO Sense 麦克风常见 GPIO41 数据 / GPIO42 时钟）
- 联网：WiFi STA
- 触发：BOOT 按钮（GPIO0）或外接按钮
- 音频格式：16kHz / 16bit / 单声道 PCM（语音够用，省带宽）
- 传输：HTTP POST 上传 WAV

### 服务器端
- 复用项目 A 的 Hermes 语音链路
- 可能需要加一个轻量中转 HTTP 服务：
  - `POST /talk`：收板子音频 → STT → Hermes → （TTS → 音响）
- 网络：v1 同局域网（板子访问服务器内网 IP），v2 可走公网 IP
