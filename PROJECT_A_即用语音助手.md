# 项目 A：即用语音助手（零代码 / 最快见效）

> 目标：满足"我要跟 Hermes 语音对话"这个**功能需求**，用最快路径实现。
> 特点：不写代码、不用 ESP32、今晚就能跑通。

---

## 一句话方案

麦克风和音响都直接插到家庭服务器上，Hermes 自带的语音能力（STT + TTS）
负责"听懂"和"说话"，一台机器搞定全部。

---

## 架构（极简）

```
🎤 USB麦克风 ──插服务器──► Hermes服务器 ──线/蓝牙──► 🔊 家庭音响
                          (STT → LLM → TTS)
```

- 麦克风：USB 直插服务器
- 服务器：Hermes 一条龙（语音转文字 → 思考 → 文字转语音）
- 音响：3.5mm 线 或 蓝牙 接服务器出声

---

## 硬件清单（全部已有，零采购）

| 设备 | 接法 |
|------|------|
| USB 麦克风 | 插服务器 USB 口 |
| 家庭音响 | 3.5mm 线 或 蓝牙 接服务器 |
| Hermes 服务器 | 家里现成的 |

---

## 为什么这个方案最优

- ✅ 零成本：所有设备都有
- ✅ 零代码：用 Hermes 现成的语音模式
- ✅ 最快：配置一下就能用
- ✅ 最稳：没有无线传输、没有自定义固件

Hermes 的语音能力参考：
- 语音模式: https://hermes-agent.nousresearch.com/docs/user-guide/features/voice-mode
- TTS: https://hermes-agent.nousresearch.com/docs/user-guide/features/tts/

安装语音依赖（服务器上）：
```bash
pip install "hermes-agent[voice]"
# 高级 TTS（可选）
pip install "hermes-agent[tts-premium]"
```

---

## 服务器到音响的连接（二选一，按方便来）

| 方式 | 优点 | 缺点 |
|------|------|------|
| 3.5mm 有线 | 零延迟、不断连、不用配对 | 要拉线 |
| 蓝牙 A2DP | 无线 | 服务器要有蓝牙、要配对、可能延迟 |

> 服务器是普通电脑/主机的话有经典蓝牙，连家里蓝牙音响没问题。
> （注意：这跟 ESP32 不一样，ESP32 才缺经典蓝牙）
